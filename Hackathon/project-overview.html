ğŸ”‘ DID Generation: Your Self-Sovereign Identity

The Magic of did:key

When you click "Generate DID", here's what happens behind the scenes:

1
Generate Ed25519 Key Pair

The browser's WebCrypto API creates a cryptographically secure Ed25519 key pair (public + private key).

2
Encode Public Key

The public key is encoded using Multibase (base58btc) and Multicodec formats.

3
Create DID String

The encoded key becomes your DID: did:key:z6MkpTHR8VNsBxYAAWHut2Geadd9jSwuBV8xRoAnwWsdvktH

4
Store Locally

Your DID and private key are stored in browser localStorage for signing operations.

// Simplified DID Generation Code const keyPair = await crypto.subtle.generateKey( { name: "Ed25519", namedCurve: "Ed25519" }, true, ["sign", "verify"] ); const publicKeyBytes = await crypto.subtle.exportKey("raw", keyPair.publicKey); const multicodecKey = new Uint8Array([0xed, 0x01, ...publicKeyBytes]); const did = "did:key:" + base58btc.encode(multicodecKey); // Result: did:key:z6MkpTHR8VNsBxYAAWHut2Geadd9jSwuBV8xRoAnwWsdvktH

Why did:key? It's self-contained (no blockchain lookup needed), instant to create, and perfect for MVP demonstrations. The public key IS the identifier!

Layer 1: Client-Side Encryption

Before files ever leave your browser:

  • AES-GCM Encryption: Military-grade 256-bit encryption using WebCrypto API
  • Random IV: Each encryption uses a unique initialization vector

// File Encryption Process const key = await crypto.subtle.generateKey( { name: "AES-GCM", length: 256 }, true, ["encrypt", "decrypt"] ); const iv = crypto.getRandomValues(new Uint8Array(12)); const encrypted = await crypto.subtle.encrypt( { name: "AES-GCM", iv: iv }, key, fileData ); // Only encrypted data is uploaded!

Layer 2: Encrypted Storage (R2)

Your files in the cloud:

  • Already Encrypted: R2 receives only encrypted data
  • No Plaintext: Even Cloudflare can't read your files
  • S3-Compatible: Industry-standard object storage
  • Access Control: Only authorized API calls can retrieve files

Layer 3: Credential Vault Encryption

Your credentials are double-encrypted:

  • Passphrase-Based: You choose a passphrase to encrypt your vault
  • PBKDF2 Key Derivation: Passphrase â†’ cryptographic key using 100,000 iterations
  • Unique Salt: Each credential has a unique salt for key derivation
  • AES-GCM Again: Credentials encrypted before localStorage storage
// Vault Encryption Process const salt = crypto.getRandomValues(new Uint8Array(16)); const keyMaterial = await crypto.subtle.importKey( "raw", new TextEncoder().encode(passphrase), "PBKDF2", false, ["deriveBits", "deriveKey"] ); const key = await crypto.subtle.deriveKey( { name: "PBKDF2", salt: salt, iterations: 100000, hash: "SHA-256" }, keyMaterial, { name: "AES-GCM", length: 256 }, true, ["encrypt", "decrypt"] ); // Credentials encrypted with derived key

Layer 4: Cryptographic Signatures

Tamper-proof credentials:

  • Ed25519 Signatures: Each credential signed by issuer's private key
  • JWT Proof Format: Industry-standard JSON Web Token proofs
  • Holder Signatures: Presentations signed by holder to prove ownership
  • Verification Without Issuer: Anyone can verify signatures using public keys

Layer 5: Blockchain Anchoring

Immutable proof of existence:

  • Hash Storage: Credential hashes (not data) stored on blockchain
  • Timestamp Proof: Blockchain provides immutable timestamp
  • No Reversal: Once anchored, cannot be altered or deleted
  • Public Verification: Anyone can verify a credential was issued at a specific time

ğŸ” Cryptographic Primitives

AES-GCM (Advanced Encryption Standard - Galois/Counter Mode)

  • Type: Authenticated encryption
  • Key Size: 256 bits
  • IV Size: 96 bits (12 bytes)
  • Authentication Tag: 128 bits
  • Use Case: File encryption, vault encryption
  • Security: Provides both confidentiality and authenticity

Ed25519 (Edwards-curve Digital Signature Algorithm)

  • Type: Public-key signature system
  • Curve: Curve25519
  • Key Size: 256 bits
  • Signature Size: 512 bits (64 bytes)
  • Use Case: DID generation, credential signing
  • Security: Resistant to timing attacks, very fast

PBKDF2 (Password-Based Key Derivation Function 2)

  • Type: Key derivation function
  • Hash: SHA-256
  • Iterations: 100,000
  • Salt Size: 128 bits (16 bytes)
  • Use Case: Deriving encryption keys from passphrases
  • Security: Slows down brute-force attacks
